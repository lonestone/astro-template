<SplitSection>
  <Fragment slot="left">
    ## Associez-le à AstroCMS

    Ce template est livré avec [AstroCMS](https://github.com/lonestone/astrocms) pré-configuré. Les éditeurs modifient le contenu via une interface web et le CMS pousse les changements MDX directement dans Git. Pas de base de données, pas de backend, pas de divergence entre staging et prod.

    Une commande suffit pour le lancer : `npm run astrocms`.
  </Fragment>

  <Fragment slot="right">
    ## Conçu et maintenu par Lonestone

    Le template est maintenu par [Lonestone](https://lonestone.io), un studio produit qui l'utilise au quotidien pour livrer des sites marketing et documentation à ses clients. Les améliorations arrivent ici au fil de nos propres sites, et vous en profitez gratuitement.

  <Fragment slot="right">
    <FaqItem question="Le template est-il libre d'utilisation ?">
      Oui. Il est sous licence MIT. Utilisable tel quel pour des projets personnels ou commerciaux.
    </FaqItem>

    <FaqItem question="Dois-je garder toutes les sections ?">
      Non. Chaque collection (blog, cas clients, docs, etc.) est optionnelle. Supprimez celles qui ne vous servent pas et le reste continue de fonctionner.
    </FaqItem>

    <FaqItem question="Comment ajouter une nouvelle langue ?">
      Ajoutez la locale dans `website.config.ts`, créez `src/content/translations/<lang>.yaml`, puis ajoutez des fichiers de contenu suffixés par `/<lang>.mdx`.
    </FaqItem>

    <FaqItem question="AstroCMS a-t-il besoin d'un backend ?">
      Non. [AstroCMS](https://github.com/lonestone/astrocms) édite directement les fichiers MDX et peut pousser les changements dans Git. Rien d'autre n'est nécessaire.
    </FaqItem>
  </Fragment>
